## ■ 開発ステップ
### STEP1：基本設計（Task / TaskLogの定義）
本アプリでは、行動を「Task」と「TaskLog」の2つの要素に分けて管理する。

#### ■ 設計方針
  
* Task : 行動の種類を管理するマスタデータ。  
**項目**
  * タスクID
  * タスク名（例：Java学習、買い物、書類作成 など）
  * カテゴリ（任意：学習・仕事・家事 など）
  
* TaskLog : 実際に行った行動の記録データ。  
**項目**
  * 実行日時
  * タスクID
  * 実行時間
  * メモ（任意）
  
以上のように役割を分離し、データの一貫性と拡張性を確保する。  

TaskとTaskLogの関係  
TaskLogはTaskのIDを保持し、どのTaskが実行されたかを示す。  
1つのTaskに対して、複数のTaskLogが紐づく（1対多の関係）。  

#### ■ 補足
当初は「行動の詳細」をTaskLogに直接記録する案も検討したが、
集計やデータの一貫性を考慮し、Taskに集約する設計とした。

---

## ■ STEP2：基本機能の実装

TaskおよびTaskLogの基本機能として、登録および一覧表示の実装を行った。
本ステップではデータの永続化は行わず、メモリ上で管理するシンプルな構成とした。

---

### ■ 実装内容

* Taskの登録
* Taskの一覧表示
* TaskLogの登録
* TaskLogの一覧表示

---

### ■ 設計上のポイント

* TaskとTaskLogを明確に分離し、それぞれの責務を限定した
* TaskLogはTaskのIDを保持することで、実行したタスクを識別する構造とした
* TaskのIDは自動採番とし、インスタンス生成時に一意の値を付与する設計とした
* IDの整合性を保つため、外部からIDを変更できないようにした
